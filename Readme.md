## Engineering Principles

Systems I build are designed with a few hard constraints in mind:

- **Ownership boundaries** — Lifecycle, validation, and aggregation concerns should remain explicitly separated
- **State management** — Operational systems should maintain deterministic and recoverable state transitions
- **Idempotency** — Retries should converge to the same system state
- **Auditability** — Every significant state change should leave a traceable record
- **Operational reliability** — Systems should degrade gracefully, surface errors clearly, and not require manual recovery for predictable failure modes
